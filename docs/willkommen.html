Willkommen zur GraphQL-Dokumentation des Mainservers der Smart Village App

Herzlich willkommen zur technischen Dokumentation des Mainservers der Smart Village App. Diese Dokumentation bietet eine umfassende Übersicht über die GraphQL-API des Mainservers, die als zentrale Schnittstelle für verschiedene Smart Village Dienste dient.

Einleitung

#

Die Ziele dieser Dokumentation sind:

  • Bereitstellung detaillierter Informationen zur Nutzung der GraphQL-API.
  • Unterstützung bei der Integration und Nutzung der API für verschiedene Anwendungen innerhalb des Smart Village Ökosystems.
  • Bereitstellung von Beispielen und Anleitungen zur effizienten Nutzung der API.

Authentifizierung

#

Die Authentifizierung erfolgt mittels OAuth2. Dazu benötigen Sie einen API-Key und ein API-Secret, um einen Bearer-Token zu generieren. Die Schritte zur Authentifizierung sind wie folgt:

  1. API-Key und -Secret erhalten: Registrieren Sie Ihre Anwendung, um die erforderlichen Zugangsdaten zu erhalten.
  2. Token anfordern: Verwenden Sie den API-Key und das API-Secret, um einen Bearer-Token anzufordern. Dies geschieht typischerweise durch einen POST-Request an den OAuth2-Token-Endpunkt.
  3. Token verwenden: Fügen Sie den erhaltenen Bearer-Token in den Authorization-Header Ihrer API-Requests ein. Stellen Sie sicher, dass der Token bei jeder Anfrage an die API beigefügt wird, um Zugriff auf die geschützten Ressourcen zu erhalten.
